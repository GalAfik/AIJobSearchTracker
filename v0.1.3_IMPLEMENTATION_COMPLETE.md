## 🎨 Features in Detail

### Feature 1: Unsaved Changes Warning

**User Experience:**
```
1. User adds/edits/deletes a job → HasUnsavedChanges = true
2. User tries to close application → Warning dialog appears
3. User chooses:
   - "Yes" → Save and close
   - "No" → Discard changes and close
   - "Cancel" → Stay in application
```

**Technical Implementation:**
- `HasUnsavedChanges` property tracks project state
- Set to `true` on any modification (add, edit, delete, import)
- Reset to `false` on save or load operations
- `CheckUnsavedChanges()` method handles dialog logic
- `MainWindow.Closing` event intercepts window close
- `e.Cancel = true` prevents close if user cancels

### Feature 2: Auto-Save After Import

**User Experience:**
```
1. User enables preference in Preferences → Appearance
2. User imports CSV into saved project
3. Project automatically saves
4. Confirmation message displays
```

**Technical Implementation:**
- New `AutoSaveAfterImport` preference property
- Checkbox in Preferences dialog
- `ImportCsv()` method checks preference
- Only auto-saves if:
  - Preference is enabled
  - Project is already saved (`_currentFilePath != null`)
- Shows success or error message
- Graceful error handling if save fails

## 💡 Implementation Notes

### Design Decisions

1. **Auto-save is opt-in**: Default is `false` to respect user control
2. **Auto-save only for saved projects**: Prevents unwanted file creation
3. **Unsaved changes is always enabled**: Cannot be disabled (data protection)
4. **Three-option dialog**: Yes/No/Cancel gives full control
5. **Clear messaging**: User always knows what's happening

### Edge Cases Handled

- ✅ Multiple unsaved changes accumulate
- ✅ Save As resets unsaved flag
- ✅ Load project resets unsaved flag
- ✅ Import into new project marks as unsaved
- ✅ Auto-save failure shows error (doesn't crash)
- ✅ Cancel SaveAs dialog keeps window open
